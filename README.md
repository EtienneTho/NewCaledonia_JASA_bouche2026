# Scripts to generate figures of BouchГ© et al. (2026) The Journal of the Acoustical Society of America

This repository contains the analysis scripts used to generate the figures for:

**BouchГ©, A., Jourdan, H., Takerkart; S., De MГ©ringo, H., Thoret, E.*, Gasc, A.* (2026). _Quantifying the effect of forest edge on tropical fauna using explainable ecoacoustics metrics._ The Journal of the Acoustical Society of America. https://doi.org/10.17605/OSF.IO/BAU9R](https://doi.org/10.1121/10.0043951**

The scripts are numbered and should be run in order. They assume that the input file `data.csv` is located in the same directory as the scripts, unless the paths are modified manually inside the scripts.

The `data.csv` file can be downloaded at the following link: [OSF repository](https://doi.org/10.17605/OSF.IO/BAU9R)

---

## Overview

The pipeline contains five numbered Python scripts:

```text
00_time_window_rsa.py
01_compute_noise_ceiling.py
02_Figure2.py
03_results_on_22pm_4am_time_range.py
04_Figure3.py
```

Although the article figures are generated by `02_Figure2.py` and `04_Figure3.py`, the preceding scripts compute the intermediate results required to build these figures.

---

## Requirements

The scripts require Python 3 and the following Python packages:

```bash
pip install numpy pandas matplotlib seaborn scikit-learn scipy statsmodels tqdm joblib
```

The main input file must be named:

```text
data.csv
```

and must contain the acoustic indices, metadata, time information, rain labels, zone identifiers, and distance-to-edge information used by the scripts.

Run the scripts from the project folder in the following order:

```bash
python 00_time_window_rsa.py

python 02_Figure2.py

python 04_Figure3.py
```

The order is important because `02_Figure2.py` reads the RSA results generated by `00_time_window_rsa.py` and the noise-ceiling results generated by `01_compute_noise_ceiling.py`.

## Script descriptions

### `00_time_window_rsa.py`

This script performs the main time-window RSA analysis across the full 24-hour cycle.

It:

- loads `data.csv`;
- removes rainy observations;
- converts distance-to-edge labels into numerical values;
- extracts and z-scores numerical ecoacoustic features;
- computes observed representational dissimilarity matrices (RDMs);
- computes model RDMs based on distance to forest edge;
- evaluates RSA as a function of sliding time windows;
- estimates confidence intervals with leave-N-zones-out bootstrap;
- estimates feature contributions with a BUBBLES-style reverse-correlation procedure;
- exports the RSA and BUBBLES results.

Main output folder:

```text
exports_rsa/
```

Important outputs include:

```text
exports_rsa/rsa_results.csv
exports_rsa/rsa_results_RSA_R2_only_reloadable.csv
exports_rsa/rsa_results_BUBBLES_weights_matrix.csv
exports_rsa/rsa_results_BUBBLES_feature_null_stats.csv
exports_rsa/rsa_results_BUBBLES_feature_pvalues_matrix.csv
exports_rsa/rsa_results_FIG_RSA_R2_only.png
```

These outputs are required by the Figure 2 script.

---

### `01_compute_noise_ceiling.py`

This script computes the time-resolved noise ceiling used to interpret the RSA results.

It:

- loads `data.csv`;
- removes rainy observations;
- extracts and z-scores numerical ecoacoustic features;
- computes day-to-day RDM reliability within each time window;
- estimates the noise ceiling using pairs of days and leave-N-zones-out resampling;
- exports the time-resolved noise-ceiling estimates.

Main output folder:

```text
exports_noise_ceiling/
```

Important outputs include:

```text
exports_noise_ceiling/noise_ceiling_results.csv
exports_noise_ceiling/noise_ceiling_results.png
```

The file `noise_ceiling_results.csv` is required by `02_Figure2.py`.

---

### `02_Figure2.py`

This script generates **Figure 2** of the paper.

It reads:

```text
exports_rsa/rsa_results.csv
exports_noise_ceiling/noise_ceiling_results.csv
```

and produces a publication-ready figure combining:

- the observed RSA time course;
- bootstrap confidence intervals;
- statistical significance markers;
- the estimated noise ceiling;
- a broken y-axis to show low and high values clearly.

Main outputs:

```text
Figure_2.eps
Figure_2.png
```

These files are exported in the same folder from which the script is launched.

---

### `03_results_on_22pm_4am_time_range.py`

This script performs a focused analysis on the nighttime window from **22:00 to 04:00**.

It:

- loads `data.csv`;
- selects the 22:00вЂ“04:00 time range;
- computes RSA for this specific time window;
- computes BUBBLES feature weights;
- compares bootstrap and null distributions;
- performs post hoc feature selection;
- exports summary tables and diagnostic figures.

Main output folder:

```text

```

Important outputs include:

```text

output/rsa_results_single_window_bootstrap_vs_null_bubbles_features.csv

output/rsa_results_single_window_bootstrap_vs_null_bubbles_selected_features.csv
output/rsa_results_single_window_bootstrap_vs_null_feature_subset_counts.csv
output/rsa_results_single_window_bootstrap_vs_null_feature_subsets_long.csv
output/rsa_results_single_window_bootstrap_vs_null_rsa_feature_sets.csv
output/rsa_results_single_window_bootstrap_vs_null_bubbles_distributions_long.csv
output/rsa_results_single_window_bootstrap_vs_null_RSA.png
output/rsa_results_single_window_bootstrap_vs_null_BUBBLES_violinplot_sorted_by_bootstrap.png
```

This script provides the detailed nighttime RSA/BUBBLES results associated with the explainable ecoacoustics analysis.

---

### `04_Figure3.py`

This script generates **Figure 3** of the paper.

It:

- loads `data.csv`;
- compares soundscape similarity between distance-to-edge groups;
- separates the analysis into two time windows:
  - `4-22h`
  - `22-4h`
- computes similarity matrices between distance-to-edge levels;
- summarizes within-near, within-far, and between-group similarities;
- runs repeated-measures ANOVA and Holm-corrected post hoc tests;
- generates a combined figure with two similarity matrices and one grouped boxplot.

Main outputs:

```text
Figure_3.eps
Figure_3.png
```

These files are exported next to the script, even if the script is launched from another working directory.

## Reproducibility notes

Several scripts define random seeds at the top of the file. This helps make the bootstrap, permutation, and resampling procedures reproducible.

Some scripts use parallel processing through `joblib` with:

```python
n_jobs = -1
```

This uses all available CPU cores. If the analysis is too demanding on a given machine, change this line to:

```python
n_jobs = 1
```

to run the computation sequentially.

## Troubleshooting

### `FileNotFoundError: data.csv`

Make sure that `data.csv` is present in the same folder as the scripts, or update the corresponding path variable in each script.

### `02_Figure2.py` cannot find RSA or noise-ceiling files

Run the following scripts first:

```bash
python 00_time_window_rsa.py
python 01_compute_noise_ceiling.py
```

Then run:

```bash
python 02_Figure2.py
```

### Long computation time

The RSA, bootstrap, BUBBLES, and noise-ceiling analyses can be computationally intensive. Runtime depends on the number of observations, zones, acoustic features, permutations, bootstrap repetitions, and CPU cores.

To reduce runtime for testing, lower parameters such as:

```python
n_boot
n_perm
n_bubble_trials
n_bubble_null
n_trials
N_REPETITIONS
```

For final publication figures, use the full parameter values reported in the scripts.
